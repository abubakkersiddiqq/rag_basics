we choose create_retrieval_chain because - this is modern one and also we can use chain type in 
separate thing not like RetrievalQA whre we use as argument and also we have control

Chain type
stuff   -- paste all chunks together, send once
              "here is everything, answer the question"

map_reduce  -- send each chunk separately, get 10 mini answers
              then send those 10 answers together, get final answer
              2 rounds of LLM calls

refine      -- send chunk 1, get answer
              send chunk 2 + previous answer, refine
              send chunk 3 + refined answer, refine again
              N rounds of LLM calls

map_rerank -- send each chunk separately, get answer + confidence score
              pick the highest confidence answer

and for the chain type we choose - stuff - we have small data also it;s best compared to the Rest
